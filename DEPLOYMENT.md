# دليل النشر والتشغيل السحابي لمنصة ذكاء الأسهم السعودية 🚀🌐

هذا الدليل يشرح خطوة بخطوة كيفية رفع وتشغيل التطبيق في بيئة سحابية حية (Production Cloud) باستخدام **Vercel** وهي المنصة الأسهل والأنسب لتطبيقات Next.js.

---

## 🛠️ الخطوة 1: تهيئة مستودع الكود (Git & GitHub)

1. افتح حسابك على [GitHub](https://github.com).
2. أنشئ مستودعاً جديداً (New Repository) واجعله **خاصاً (Private)** لحماية كود المنصة وقاعدة البيانات.
3. افتح منفذ الأوامر في مجلد المشروع المحلي وقم برفع الكود باستخدام الأوامر التالية:
   ```bash
   git init
   git add .
   git commit -m "Initial production commit"
   git branch -M main
   git remote add origin <رابط_مستودعك_على_غيتهاب>
   git push -u origin main
   ```

---

## ⚡ الخطوة 2: النشر والتشغيل على Vercel

1. اذهب إلى موقع [Vercel](https://vercel.com) وسجل الدخول باستخدام حساب GitHub الخاص بك.
2. اضغط على زر **"Add New"** ثم اختر **"Project"**.
3. ستظهر لك قائمة بمستودعاتك على GitHub، ابحث عن مستودع مشروع **saudistock** واضغط على **"Import"**.
4. في شاشة إعداد المشروع، ستجد قسماً هاماً جداً يسمى **"Environment Variables"** (متغيرات البيئة). قم بنسخ وإضافة المتغيرات التالية بدقة:

### 🔑 متغيرات البيئة المطلوبة في Vercel:

| اسم المتغير (Key) | القيمة (Value) | الوصف |
| :--- | :--- | :--- |
| `DATABASE_URL` | `postgresql://neondb_owner:npg_V4nO1CpYaiEX...` | رابط قاعدة بيانات Neon Postgres (انسخه من ملف `.env` المحلي) |
| `NEXTAUTH_SECRET` | `اكتب_هنا_نصاً_عشوائياً_وقوياً` | رمز سري لتشفير جلسات تسجيل الدخول وتأمينها |
| `NEXTAUTH_URL` | `https://your-app-name.vercel.app` | الرابط الذي سيمنحه لك Vercel (أو رابط الدومين الخاص بك) |

*💡 ملاحظة: لقد قمنا بتعديل ملف `package.json` ليقوم تلقائياً بتشغيل أمر `prisma generate` أثناء بناء المشروع على Vercel لضمان اتصال قاعدة البيانات دون أي تدخل يدوي منك.*

5. اضغط على زر **"Deploy"**. سيقوم Vercel ببناء المشروع فوراً وتشغيله حياً على الإنترنت.

---

## 💾 الخطوة 3: قاعدة البيانات السحابية (Neon Postgres)

ميزة قوية جداً في هذا التطبيق هي أن قاعدة البيانات مستضافة بالكامل على سحابة **Neon Cloud (AWS AWS London)**. 
* **لا تحتاج إلى نقل أو إعداد قاعدة بيانات جديدة!**
* سيرفر Vercel الجديد سيتصل مباشرة وبشكل آمن بقاعدة البيانات السحابية الحالية.
* في حال قمت مستقبلاً بتعديل على جداول قاعدة البيانات (Schema)، كل ما عليك فعله هو تشغيل الأمر التالي محلياً لتحديث السحابة:
  ```bash
  npx prisma db push
  ```

## 🔒 تأمين وحماية الجلسات في الإنتاج (Production Security)

عند تشغيل التطبيق في البيئة السحابية الحية، يتأكد نظام الحماية NextAuth من تشفير ملفات الكوكيز (Cookies) عبر بروتوكول HTTPS تلقائياً لتأمين حسابات المشتركين ومنع أي اختراق للجلسات.
